# 프론트엔드 구현 가이드

## 📦 구현 완료 내용

### 1. 로비 시스템
- ✅ 메인 메뉴 (방 만들기 / 방 참여하기)
- ✅ 방 생성 화면 (닉네임 입력)
- ✅ 방 참여 화면 (닉네임 + 방 코드 입력)
- ✅ API 연동 (방 생성/참여)

### 2. 게임 화면
- ✅ 게임 보드 (6개 위치 노드 + 연결선)
- ✅ 플레이어 정보 패널 (돈, 위치, 결심 토큰, 특성 점수)
- ✅ 손패 카드 표시
- ✅ 행동 버튼 (1~6번)
- ✅ 공동 계획 섹션
- ✅ 메시지 바 (현재 상태 표시)
- ✅ Day 카운터 (1~14일)

### 3. 실시간 통신
- ✅ Socket.io 클라이언트 설정
- ✅ WebSocket 이벤트 리스너
  - `turn-started`: 턴 시작 알림
  - `state-updated`: 게임 상태 업데이트
  - `chance-request`: 찬스 카드 상호작용 요청

### 4. API 서비스
- ✅ Axios 기반 API 클라이언트
- ✅ 모든 게임 API 엔드포인트 연동
- ✅ Vite 프록시 설정 (CORS 해결)

## 🎨 UI/UX 특징

### 디자인
- 그라데이션 배경 (보라색 계열)
- 카드 기반 레이아웃
- 반응형 디자인 (모바일 대응)
- 부드러운 애니메이션 효과

### 색상 팔레트
- Primary: `#667eea` (보라)
- Secondary: `#764ba2` (진한 보라)
- Success: `#27ae60` (초록)
- Danger: `#e74c3c` (빨강)

### 컴포넌트 구조
```
App
├── LobbyScreen (로비)
│   ├── 메인 메뉴
│   ├── 방 만들기
│   └── 방 참여하기
└── GameScreen (게임)
    ├── GameBoard (게임 보드)
    ├── PlayerInfo (플레이어 정보)
    ├── HandCards (손패)
    └── JointPlan (공동 계획)
```

## 📝 주요 기능 구현 상태

### ✅ 완료
1. **로비 시스템**
   - 방 생성/참여
   - 닉네임 입력
   - 방 코드 공유

2. **게임 보드**
   - 6개 위치 시각화
   - 인접 칸 하이라이트
   - 클릭 이동

3. **플레이어 정보**
   - 돈, 위치, 결심 토큰
   - 7가지 특성 점수 표시

4. **턴 시스템**
   - 턴 시작 알림
   - 행동 선택
   - 턴 종료

### ⚠️ 부분 구현
1. **손패 카드**
   - UI는 완성
   - 실제 카드 데이터 연동 필요

2. **공동 계획**
   - 기본 UI 완성
   - 기여 모달 필요

3. **찬스 카드 상호작용**
   - 이벤트 수신 구조 완성
   - 상호작용 모달 필요

### ❌ 미구현
1. **대기실**
   - 플레이어 목록 표시
   - 방장 시작 버튼
   - 실시간 참여자 업데이트

2. **찬스 카드 모달**
   - C10: 구매 요청 모달
   - C11: 교환 선택 모달
   - C13: 플레이어 선택 모달

3. **최종 결과 화면**
   - 점수 표시
   - 순위 표시
   - 세부 내역
   - 소프트 리셋 버튼

4. **공동 계획 기여 모달**
   - 금액 입력
   - 현황 표시

```tsx
// ChanceModal.tsx

- 플레이어 선택 (C8, C9, C13)
- 카드 선택 (C10, C11)

// ResultScreen.tsx

- 점수 세부 내역
- 구매한 카드 목록
- 다시하기 버튼
```

### 4. 공동 계획 모달 (우선순위: Medium)
```tsx
// ContributeModal.tsx
- 금액 입력 슬라이더
- 현재 기여 현황
- 목표 금액 진행바
- 기여 버튼
```

```tsx
// useGameState.ts
- Context API 또는 Zustand
- 전역 상태 관리
- WebSocket 이벤트 통합
```

## 🐛 알려진 이슈

1. **게임 상태 동기화**
   - 현재 로컬 상태만 사용
   - 서버 상태와 동기화 필요

2. **에러 처리**
   - 기본 alert 사용
   - Toast 알림 시스템 필요

3. **로딩 상태**
   - 일부 API 호출에만 적용
   - 전역 로딩 인디케이터 필요

4. **재연결 처리**
   - WebSocket 끊김 시 재연결 로직 없음
